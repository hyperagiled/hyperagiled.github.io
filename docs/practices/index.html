프랙티스

AI-First 백로그 정제​

  • 입력: 아이디어/요구사항/버그 리포트
  • 출력: 사용자 스토리, 수용 기준, 테스트 아이디어

권장 포맷(예시)

  • 목적
  • 사용자/상황
  • 수용 기준
  • 반례/엣지 케이스
  • 측정할 메트릭

PR을 작게 유지하기​

  • 하나의 PR은 “하나의 이유”로 합칠 수 있어야 합니다.
  • 변경 범위를 줄이기 위해
    • 기능 플래그
    • 점진적 리팩터링
    • 테스트 우선

정의(DoD)를 살아있는 규칙으로​

  • 테스트 통과
  • 린트/정적 분석 통과
  • 관측 가능성(로그/메트릭) 포함
  • 롤백/릴리스 노트 포함
